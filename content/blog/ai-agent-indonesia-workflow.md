# AI Agent 工作流设计方法

## 面向印尼市场的自动化业务架构与落地模型

从流程设计出发，构建可执行的 AI Agent 系统，适用于地产、跨境服务与工业园招商场景。

## 顶层架构

- **Task Layer（任务层）**：线索收集、客户分层、销售触达等业务任务拆分
- **Orchestration Layer（编排层）**：规则、队列、状态管理与异常回滚
- **Tool Layer（工具层）**：LLM、表单系统、CRM、邮件、数据抓取引擎

## 适用业务场景

- 印尼工商业地产招商与客户筛选
- 跨境服务的自动化线索获取与评分
- 工业园区招商流程标准化与执行加速

## 落地建议

1. 先定义核心转化路径，再设计自动化节点
2. 建立可观测日志与人工兜底机制
3. 将结果回写 CRM，形成持续优化闭环
